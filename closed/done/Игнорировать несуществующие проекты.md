Сейчас при указании в todo.toml настроек вида
```toml
[project.prjA]
path = "path/prjA"

[project.prjB]
path = "other/prjB"
```
когда путь "other/prjB" не существует, то вызов `todo tree` и `todo list` завершается с ошибкой: `Error: wrong project path or configuration: project id: prjB, path: Some("other/prjB")`.
Необходимо чтобы в подобной ситуации команды `tree` и `list` всё равно выполнялись для других проектов, пути к которым указаны верно и существуют, а для проектов с несуществующими путями вывдилось сообщение об ошибке.
Нужно внести исправление в код и добавить новые test-cases в тесты `cli/tests/list.md` и `cli/tests/tree.md`.
