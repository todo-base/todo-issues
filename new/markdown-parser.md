# Перейти на готовый markdown-парсер в `tracker-fs`

Можно реализовать реальную потоковую обработку:
1. в [mdstream] построково загружаем файл, он выдаёт готовый блок;
2. сохраняем индексы начальной и конечной строк;
3. разбираем при помощи обычного парсера.

## Подходящие парсеры без потоковой обработки

> NOTE: Хорошие zero-copy крэйты с индексами в исходных данных
> (без поддержки реальной потоковой обработки из файла)

- [markdown] - парсит буфер в древовидную структуры.

- [pulldown-cmark] - итераторная обработка исходоного буфера.
  Итератор выдаёт готовые "события" (единицы документа).

[markdown]: https://crates.io/crates/markdown
[pulldown-cmark]: https://crates.io/crates/pulldown-cmark
[mdstream]: https://crates.io/crates/mdstream
