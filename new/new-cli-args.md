## Доработки `list` & `tree`

Нужно добавить пагинацию при отображении. Лучше всего подгружать view-файлы постепенно.

Порядковый номер на странице должен начинаться с единицы для каждой новой страницы.
Их можно использовать для [выбора пользователем](#возможность-что-то-делать-с-задачами).
Т.о. это *не* порядковый id, который у нас сейчас назначается при составлении списка задач.

Это будет актуально, например, в следующих случаях.

### Потенциально огромные файлы

  Юзеры могут весть один view-файл TODO.md в обратном порядке:
  добавлять все задачи сверху разделяя их на milestone.
  Это простой режим не требующий доп. нагрузки на обслуживание
  и может быть весьма популярным (особенно у ai-агентов).

### Возможность что-то делать с задачами

  Например, можно добавить обработку клавиш и внизу добавить строку состояния с подсказками:
  `[e]dit, [d]escription, [o]pen, [f]ilter ...`.

  Либо vim-мод с командами через `:` и горчии клавиши.

  При нажатии на `e` можно предложить выбрать задачу по порядковому номеру на текущей странице.
